    **BUG-01 (per D-01 — CONTEXT.md BUG-01 decision):**
    In `bot/commands/addtheme.py`, replace `_save_custom_theme` function (lines 81-98):

    Current (buggy):
    ```python
    db.execute(
        "INSERT INTO custom_themes (user_id, name, hashtag, rss_feeds, ai_suggested) "
        "VALUES (?, ?, ?, ?, ?)",
        [user_id, name, hashtag, json.dumps(rss_feeds), 1 if ai_suggested else 0],
    )
    rows = db.execute("SELECT last_insert_rowid() as id", [])
    custom_id = rows[0]["id"]
    ```

    Replace with:
    ```python
    rows = db.execute(
        "INSERT INTO custom_themes (user_id, name, hashtag, rss_feeds, ai_suggested) "
        "VALUES (?, ?, ?, ?, ?) RETURNING id",
        [user_id, name, hashtag, json.dumps(rss_feeds), 1 if ai_suggested else 0],
    )
    custom_id = rows[0]["id"]
    ```

    This collapses two queries into one atomic operation. The `db.execute()` call with `RETURNING id` returns `[{"id": <new_id>}]`. Keep the rest of the function (the `execute_many` for `user_themes` INSERT) unchanged.

    Update the comment on the line above to: `# Atomic INSERT with RETURNING avoids TOCTOU race`

    **BUG-03 (per D-03 — CONTEXT.md BUG-03 decision):**
    In `bot/commands/payments.py`, replace lines 37-38 in `handle_successful_payment`:

    Current (crashes on malformed payload):
    ```python
    payload = payment["invoice_payload"]  # e.g. "tier:one_time"
    tier = payload.split(":", 1)[1]
    ```

    Replace with:
    ```python
    payload = payment["invoice_payload"]  # e.g. "tier:one_time"
    if ":" not in payload:
        logging.error("handle_successful_payment: malformed payload %r for user %d", payload, user_id)
        tg.send_message(
            chat_id=user_id,
            text="⚠️ Payment received but could not be processed. Please contact support.",
        )
        return
    tier = payload.split(":", 1)[1]
    ```

    This validates the payload contains `:` before splitting, preventing `IndexError`. On malformed payload: logs error with raw payload, sends user-facing error message, returns early (no DB write).
